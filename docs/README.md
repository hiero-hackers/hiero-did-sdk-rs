# Docs

Focused guides for working with this SDK workspace.

## Guides

- [`create-did.md`](./create-did.md): DID creation flows and examples.
- [`api-reference.md`](./api-reference.md): public API surface by crate.
- [`testing.md`](./testing.md): local and integration test setup.

## Recommended Read Order

1. Start with `create-did.md` if you want end-to-end DID creation.
2. Use `api-reference.md` while implementing against specific crates.
3. Read `testing.md` before running networked tests.
